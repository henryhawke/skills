# Incident Response & SRE

## When to use
- Respond to production incidents
- Troubleshoot system failures
- Write blameless post-mortems
- Create incident runbooks
- Manage on-call handoffs
- Implement SRE practices

## Incident workflow
1. Detect and alert
2. Triage severity and impact
3. Assemble response team
4. Investigate and diagnose
5. Mitigate and resolve
6. Communicate status
7. Post-mortem and action items

## Troubleshooting
- Log analysis and correlation
- Distributed tracing investigation
- Kubernetes pod/node debugging
- Network connectivity diagnosis
- Database deadlock/slowness analysis
- Memory leak detection

## SRE practices
- SLI/SLO/SLA definition
- Error budget management
- Toil reduction
- Capacity planning
